## 一、产品概述

### 1.1 产品定位

一个面向学生和上班族的**低电量恢复社区**。不贩卖焦虑，不逼迫自律，而是用“摆烂”的语言体系，通过**情绪共鸣→认知许可→微行动启动→社区陪伴**的路径，帮助用户从“完全宕机”的状态中轻轻爬起来一点点。

### 1.2 目标用户

| 用户群 | 核心特征 | 核心痛点 |
| :--- | :--- | :--- |
| 学生 | 大学生、研究生为主，考研/考公党 | 论文拖延、考试焦虑、未来迷茫、白天摆烂晚上愧疚 |
| 上班族 | 初入职场1-5年，城市白领为主 | 工作被消耗、下班瘫痪、想努力没劲、需要微小掌控感 |

### 1.3 核心价值主张

- **对用户：** 提供一个“被理解”的互联网情绪避难所。在这里，“摆烂”不是失败，而是一种被允许的自我保护。通过极低成本的行动，重拾微小的掌控感。
- **对市场：** 切入“情绪陪伴与微行动恢复”这一细分领域，用高共鸣、强传播的内容（摆烂许可证）作为流量入口，构建高粘性年轻用户社区。

### 1.4 MVP核心目标

| 指标类型 | 具体指标 | 目标值 |
| :--- | :--- | :--- |
| 核心功能验证 | 摆烂许可证生成率 | 进入用户中 > 60% 完成生成 |
| 传播模式验证 | 卡片分享回流率 | 从分享卡片进入小程序的比例 > 15% |
| 初步留存验证 | 次日留存 / 7日留存 | 次日 > 20%，7日 > 10% |

## 二、用户场景

### 2.1 学生典型场景

#### 场景一：论文启动困难

> 小陈，大四。毕业论文DDL逼近，一个字没写。白天刷手机麻痹自己，晚上躺在床上被愧疚感淹没。他打开「今天也没废」，选择状态“论文装死型”。系统生成一张“论文难产诊断书”，并给出任务：“打开文档，把标题打上去。完成后你可以继续摆烂，但论文已经不是0了。”他照做了，焦虑感明显下降，顺势写了第一段。

#### 场景二：考试周崩溃

> 考研倒计时30天，小林感觉脑子一片空白。她在“考研低电量小组”里看到无数状态相似的同伴，用“复习不动但还想抢救一下”的方式互相打气。她点击“只背3个关键词”任务，完成后获得一张“考前临时做人许可证”，分享到考研群，收获一片“我也是”。

### 2.2 上班族典型场景

#### 场景一：下班后的精神回血

> 小李，工作2年。经历了一天无意义会议和领导批评后，通勤地铁上只想刷短视频。她打开产品，选择状态“老板创伤型”，生成“打工人低电量许可证”，许可条款是“下班后不复盘人生，只洗澡”。她到家后完成了附带的微任务“把外卖盒扔掉”，感觉恢复了一点对生活的掌控。

#### 场景二：工位摸鱼修复

> 下午4点，会议连轴转之后精神涣散。小王使用“3分钟摸鱼修复”，产品生成一张“工位灵魂出逃报告”卡片。他笑着转发到同事小群，几个人开始讨论“你今天的精神电量剩多少”，成为当天办公室的解压话题。

## 四、MVP功能详细设计

### 功能一：双模式身份选择

#### 4.1.1 功能描述

用户首次进入产品时，选择“学生模式”或“上班族模式”。选择后首页主题、状态库、社区分区均随之切换。用户可在设置中随时切换模式。

#### 4.1.2 页面元素

- 两个大卡片选项，各配一句共鸣文案
- 学生模式卡片文案：“论文、考试、考研、迷茫——你不是一个人”
- 上班族模式卡片文案：“会议、KPI、加班、通勤——电量低不是你的错”

#### 4.1.3 交互逻辑

- 选择后进入对应模式首页
- 选择记录缓存在本地，下次打开直接进入
- 底部导航栏保留“切换模式”入口

---

### 功能二：今日状态选择器

#### 4.2.1 功能描述

用户从预设状态库中选择最能描述当前感受的一个状态。这是生成许可证的核心输入项。

#### 4.2.2 状态库完整定义

**学生模式状态库：**

| 状态名称 | 情绪关键词 | 适用场景 |
| :--- | :--- | :--- |
| 论文装死型 | 拖延、逃避、愧疚 | 论文/大作业DDL逼近但无法启动 |
| 作业积压型 | 焦虑、无力、压力 | 多项作业堆积，不知从何下手 |
| 考前装死型 | 紧张、逃避、麻木 | 考试临近但复习不进去 |
| 考研低电量型 | 疲惫、迷茫、孤独 | 长期备考，精神电量持续走低 |
| 上课灵魂出走型 | 无聊、游离、机械 | 人在课堂，心在宇宙 |
| 明明很急但就是不动型 | 焦虑、矛盾、自我谴责 | 知道该做但身体拒绝执行 |

**上班族模式状态库：**

| 状态名称 | 情绪关键词 | 适用场景 |
| :--- | :--- | :--- |
| 会议吸干型 | 疲惫、麻木、被消耗 | 连续开会，精神被榨干 |
| 老板创伤型 | 委屈、愤怒、无力 | 被批评/否定后需要恢复 |
| KPI压迫型 | 焦虑、压力、窒息感 | 业绩压力大，喘不过气 |
| 通勤废掉型 | 疲惫、机械、麻木 | 长时间通勤耗尽全部能量 |
| 下班瘫痪型 | 瘫软、放空、不想动 | 到家后完全无法启动任何事 |
| 想辞职但没钱型 | 矛盾、被困住、憋屈 | 想逃离但现实不允许 |

#### 4.2.3 交互逻辑

- 状态卡片以网格或列表形式展示，每个状态配一个小icon
- 选中后有轻微动效（如“确认当前精神状态”的勾选动画）
- 选中后自动跳转至许可证生成页

### 功能三：摆烂许可证生成器（核心功能）

#### 4.3.1 功能描述

根据用户选择的身份和状态，生成一张高共鸣、可分享的情绪卡片。这是产品核心流量引擎。

#### 4.3.2 卡片标准信息架构

每张卡片包含以下7个信息模块：

| 模块 | 说明 | 是否可自定义 | 示例 |
| :--- | :--- | :--- | :--- |
| 标题区 | 证书名称，醒目展示 | 根据状态自动匹配 | “今日摆烂许可证” |
| 编号区 | 唯一证书编号，增加正式感 | 自动生成 | “BL-0525-404” |
| 身份区 | 持证人身份标识 | 自动匹配+手动修改 | “持证人：论文失联人员” |
| 原因区 | 摆烂原因诊断文案 | 根据状态自动生成 | “因论文未响应、DDL逼近...” |
| 许可区 | 今日被“批准”的事项 | 根据状态自动生成 | “批准：今日可低强度存在” |
| 任务区 | 一个极小的行动建议 | 自动匹配+手动换 | “最低行动：打开文档3分钟” |
| 金句区 | 点睛之笔，传播核心 | 根据状态自动匹配 | “你不是废，你只是电量不足” |

#### 4.3.3 MVP三套卡片视觉风格

**风格一：官方公文风**

- **适用人群：** 最通用，上班族和学生都适合
- **视觉描述：** 米白/浅灰仿旧纸张底，红章、黑体标题、细边框，严肃排版，像真正的红头文件
- **核心吸引力：** 反差感——越正经越好笑
- **分享优势：** 最强，适合所有社交平台

**风格二：精神诊断书风**

- **适用人群：** 学生、考研党、有焦虑情绪的上班族
- **视觉描述：** 白底蓝灰线条，表格、指标项、诊断意见区，像体检报告
- **核心吸引力：** “太像我了”——数值化呈现精神状态的共鸣
- **分享优势：** 适合截图式传播，像真实的诊断报告

**风格三：游戏状态面板风**

- **适用人群：** 年轻用户、二次元用户、学生群体
- **视觉描述：** 深色底、状态条、Buff/Debuff图标、技能项，像RPG角色面板
- **核心吸引力：** 年轻化、差异化、有趣
- **分享优势：** 适合年轻用户社交圈

### 功能四：AI 摆烂搭子

#### 4.4.1 功能描述

一个轻量级AI对话功能。AI不讲大道理，只做三件事：接住情绪、把用户的话变成好笑的嘴替、给一个低压力行动建议。

#### 4.4.2 对话设计规范

AI回复需遵循“三段式”结构：

| 步骤 | 动作 | 示例 |
| :--- | :--- | :--- |
| 第一步 | 接住情绪（共情） | “被当成空气攻击了是吧，这感觉太伤了。你的精神护盾已经碎了。” |
| 第二步 | 嘴替生成（幽默化） | “你现在的干劲，约等于领导画的饼那么大。” |
| 第三步 | 微行动建议（降低门槛） | “现在不需要做任何大事，只做一件事：把水杯拿过来，喝一口。” |

#### 4.4.3 AI行为约束

- **禁止说教：** 不能说“你应该……”
- **禁止打鸡血：** 不能说“加油，你一定可以的！”
- **禁止否定用户感受：** 不能说“这有什么大不了的”
- **必须用“摆烂语言体系”回应用户：** 详见附件A：文案风格指南

### 功能五：低能量社区

#### 4.5.1 功能描述

一个轻互动、高共鸣的内容社区。不鼓励长篇输出，而是用按钮、短句、卡片进行互动。核心调性是“互联网避难所”。

#### 4.5.2 内容类型

| 类型 | 说明 | 优先级 |
| :--- | :--- | :--- |
| 许可证打卡 | 用户完成微任务后，一键将证书发布到社区 | P0 |
| 共鸣卡片 | 用户分享“今日精神状态”简短描述 | P1 |
| 主题广场 | 按主题分区的讨论（如“论文难产互助会”“工位摸鱼艺术展”） | P1 |

#### 4.5.3 互动按钮设计

社区不设传统的“评论”输入框，改为预设的轻互动按钮：

| 按钮 | 含义 | 场景 |
| :--- | :--- | :--- |
| 抱一下 | 表达共情和支持 | 看到有人状态很差时 |
| 我也这样 | 表达共鸣 | 看到和自己一样的状态时 |
| 今天算了 | 表达“放过自己”的许可 | 看到有人在自我谴责时 |
| 明天再说 | 表达“不必急”的安慰 | 看到有人在焦虑时 |

#### 4.5.4 主题分区

**学生区：**

- 宿舍摆烂排行榜
- 考研低电量小组
- 论文难产互助会
- 考前临时做人广场

**上班族区：**

- 工位摸鱼艺术展
- 下班回血指南
- 职场树洞（非负能量版）
- 周一不适互助会

### 功能六：朋友互证摆烂（裂变功能）

#### 4.6.1 功能描述

用户生成许可证后，可发送给微信好友请求“批准”。好友点击后可为TA盖章，并一键生成自己的许可证。这是产品的核心裂变机制。

#### 4.6.2 流程设计
用户A生成许可证
↓
点击“让朋友批准我摆烂”
↓
分享卡片给微信好友/群
↓
好友B点击卡片进入小程序
↓
看到A的许可证 + 盖章按钮
↓
好友B选择盖章：【批准】【我也一样】【抱一下】
↓
用户A收到服务通知：“你的摆烂申请已被3位好友批准”
↓
好友B被引导生成自己的许可证（裂变闭环）
#### 4.6.3 盖章按钮设计

| 盖章 | 文案 | 效果 |
| :--- | :--- | :--- |
| 批准 | “批准摆烂，今日生效” | 红色圆章动效 |
| 我也一样 | “确认过眼神，是同一种废” | 蓝色圆章动效 |
| 抱一下 | “远程抱抱已发送” | 绿色圆章动效 |

## 五、UI/UX设计规范

### 5.1 卡片生成技术方案

#### 5.1.1 MVP技术选型：纯模板渲染

| 维度 | 方案说明 |
| :--- | :--- |
| 实现方式 | 前端Canvas渲染预设模板，用户数据（昵称、状态、任务、金句）动态填入 |
| 模板数量 | 3套风格 × 每套2个变体（学生/上班族）= 6个核心模板 |
| 生成速度 | < 500ms，所见即所得 |
| 开发成本 | 中等，需前端Canvas开发 |

#### 5.1.2 V2升级方案：AI生成背景 + 前端合成文字

| 维度 | 方案说明 |

### 5.2 UI设计原则

#### 5.2.1 核心设计哲学：四个“一眼”

| 原则 | 说明 | 设计要求 |
| :--- | :--- | :--- |
| 一眼能懂 | 分享到朋友圈，别人1秒内知道这是什么 | 标题字号最大，金句次之 |
| 一眼像“我” | 用户觉得这张卡是自己的 | 昵称、状态、身份标签必须可定制 |
| 一眼好笑 | 严肃外壳+摆烂内容的反差 | 配色正经，文案摆烂 |
| 一眼想发 | 用户不觉得发出去丢人 | 品牌露出要轻，不能像广告 |

## 七、文案风格指南（附件A）

### 7.1 品牌语言体系

**核心原则：**

- **允许但不鼓励：** 允许用户摆烂，但永远留一个“微行动”的钩子
- **共情不说教：** 说“我懂你”而不是“你应该”
- **幽默不轻浮：** 用玩笑化解焦虑，不嘲笑用户的困境
- **低能量不低气压：** 氛围是“柔软的避难所”而不是“丧气聚集地”

**禁用词汇：**

- 加油、你一定可以、相信自己
- 努力、奋斗、拼搏
- 你看看别人、别人都能做到
- 这有什么难的、有什么大不了的

**推荐词汇体系：**

- 允许、批准、今天算了、明天再说
- 抱一下、我也这样、你不是一个人
- 低电量、缓冲中、加载中、保护模式
- 烂中带救、轻轻爬起来、别彻底掉线

### 7.2 关键触点文案库

| 触点 | 学生版文案 | 上班族版文案 |
| :--- | :--- | :--- |
| 首页标题 | 今天也没废 | 今天也没废 |
| 首页副标题 | 不是戒掉摆烂，是防止你烂穿地心 | 不是让你立刻变好，而是让你别彻底掉线 |
| 空状态 | 大家都在烂，你并不孤单 | 低电量是人类正常状态，欢迎归队 |
| 任务完成 | 完成了最低行动，今天已经不亏 | 能完成这件事，今天就已经是有效人类了 |
| 分享引导 | 发给朋友，让他们批准你今天的摆烂 | 测测你同事今天是哪种摆烂人格 |
| 次日回访 | 昨天你完成了“打开文档”，今天要继续抢救吗？ | 新的一天，新的低电量，要来领取今日证明吗？ |

## 十、风险与应对

| 风险 | 影响 | 应对策略 |
| :--- | :--- | :--- |
| 卡片分享被微信判定为诱导分享 | 传播链路中断 | 不在卡片上放明显二维码，品牌落款轻量化；分享引导文案避免诱导性词汇 |
| 社区出现真实负能量/抑郁倾向内容 | 社区氛围恶化，安全风险 | 建立内容审核机制；设置心理援助热线入口；社区互动只设正向按钮 |
| 用户快速审美疲劳 | 留存下降 | 每日机制持续更新内容；V2接入AI生成增加变化性；节点运营（考试周/年终等） |
| AI生成图片效果不可控 | 用户满意度低 | MVP用纯模板渲染保证基础体验；V2 AI生成作为可选功能非默认；提示词持续优化 |
